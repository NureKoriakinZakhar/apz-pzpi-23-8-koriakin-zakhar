3 ОПИС ВИКОНАНОЇ РОБОТИ
3.1 Загальні відомості про систему
Назва системи: Netflix.
Розробник: Netflix, Inc.
Призначення системи: Глобальна стрімінгова платформа для надання відеоконтенту (фільмів, серіалів) за запитом (VOD). Цільова аудиторія охоплює понад 200 мільйонів користувачів по всьому світу. 
Основні функції: реєстрація та білінг, персоналізовані рекомендації, транскодування відео, адаптивний стрімінг контенту.


3.2 Архітектурний стиль системи
Система використовує Мікросервісну архітектуру у поєднанні з хмарною інфраструктурою. 
Обґрунтування вибору: Історичний перехід від моноліту до мікросервісів був викликаний необхідністю усунення єдиної точки відмови та забезпечення незалежного горизонтального масштабування. Це дозволяє тисячам розробників паралельно оновлювати окремі сервіси без зупинки роботи всієї платформи.


3.3 Основні компоненти системи
Опис підсистем:
* Інтерфейс користувача: Додатки для Smart TV, iOS, Android та веб-браузерів, які адаптують UI під конкретний пристрій.
* Серверна частина: API Gateway (Zuul), що приймає та маршрутизує всі вхідні запити.
* Рівень бізнес-логіки: Сотні незалежних мікросервісів на AWS (сервіс авторизації, сервіс профілів, рушій рекомендацій, білінг).
* Рівень зберігання даних: Гетерогенні бази даних (Cassandra для великих обсягів запису, Redis/Memcached для кешування, MySQL для специфічних транзакцій).
* Допоміжні сервіси: Open Connect (власна CDN для доставки важкого відеоконтенту), системи Service Discovery (Eureka), інструменти Chaos Engineering.


3.4 Взаємодія компонентів
Клієнтський пристрій відправляє запит до балансувальника навантаження (AWS ELB), який передає його до API Gateway. Шлюз звертається до сервісу Eureka, щоб дізнатися IP-адресу потрібного мікросервісу (наприклад, сервісу рекомендацій). Мікросервіс обробляє запит, звертаючись до кешу або бази даних (Cassandra), і повертає JSON клієнту. Коли клієнт натискає "Play", запит на сам відеофайл йде не на AWS, а безпосередньо до найближчого сервера Open Connect (CDN), минаючи бізнес-логіку.


  

Рисунок 1.1 – Діаграма послідовності взаємодії користувача з системою


Діаграма ілюструє покроковий потік даних під час типового сеансу користувача. Вона чітко демонструє розділення відповідальності: легкі запити (метадані, авторизація) проходять через API Gateway та кластер мікросервісів, тоді як важкий відеотрафік обслуговується виключно через глобальну мережу доставки контенту (CDN), що знімає навантаження з основних серверів AWS.


  

Рисунок 1.2 – Діаграма компонентів архітектури Netflix


Рисунок відображає високорівневу структурну організацію системи. Показано єдину точку входу для бізнес-запитів (API Gateway), використання реєстру сервісів (Eureka) для динамічної маршрутизації та патерн «Polyglot Persistence», де різні мікросервіси використовують різні типи сховищ даних (Cassandra для постійного зберігання, Redis для швидкого кешування) залежно від своїх специфічних потреб.


3.5 Масштабованість та надійність
* Механізми балансування навантаження: AWS Elastic Load Balancing та внутрішні балансувальники (Ribbon).
* Відмовостійкість: Патерн Circuit Breaker (Hystrix) для ізоляції збоїв мікросервісів; тестування надійності за допомогою Chaos Monkey.
* Кешування: Багаторівневий кеш (EVCache) для зберігання персональних даних у пам'яті.
* Реплікація: Дані дублюються у кількох географічних зонах AWS (Availability Zones).
* Хмарна інфраструктура: Повна залежність від еластичності AWS (EC2, S3) для динамічного виділення ресурсів. 


3.6 Приклади програмної реалізації
Для наочної демонстрації принципів мікросервісної архітектури та механізмів забезпечення відмовостійкості було згенеровано навчальний програмний код. Основна увага приділялася патерну Circuit Breaker (Запобіжник), який є критично важливим компонентом в інфраструктурі Netflix для запобігання каскадним збоям.


Запит до ШІ: "Згенеруй базовий приклад того, як працює API Gateway у поєднанні з Circuit Breaker (на прикладі Java/Spring Boot), який би демонстрував ізоляцію збоїв мікросервісу при взаємодії між компонентами".


1  import org.springframework.beans.factory.annotation.Autowired;
2  import org.springframework.web.bind.annotation.*;
3  import org.springframework.web.client.RestTemplate;
4  import io.github.resilience4j.circuitbreaker.annotation.CircuitBreaker;
5  import java.util.Arrays;
6  import java.util.List;
7  
8  @RestController
9  public class MovieCatalogService {
10 
11     @Autowired
12     private RestTemplate restTemplate;
13 
14     // Метод звертається до іншого (зовнішнього) мікросервісу
15     @GetMapping("/catalog/{userId}")
16     @CircuitBreaker(name = "recommendationService", fallbackMethod = "getFallbackCatalog")
17     public List<String> getCatalog(@PathVariable String userId) {
18         // Імітація виклику мікросервісу рекомендацій через мережу
19         return restTemplate.getForObject("http://recommendation-service/api/" + userId, List.class);
20     }
21 
22     // Fallback метод, який автоматично викликається при падінні recommendation-service
23     public List<String> getFallbackCatalog(String userId, Exception e) {
24         // Повертаємо закешований або дефолтний список контенту замість критичної помилки 500
25         return Arrays.asList("Базовий фільм 1", "Базовий фільм 2");
26     }
27 }


Наведений клас MovieCatalogService здійснює мережевий виклик до зовнішнього сервісу рекомендацій. Завдяки застосуванню анотації @CircuitBreaker (рядок 16), у разі недоступності цільового сервісу або перевищення часу очікування (таймауту), виняток безпечно перехоплюється системою. Замість того, щоб повернути клієнту фатальну помилку і заблокувати роботу поточного вузла, трафік миттєво перенаправляється на резервний метод getFallbackCatalog (рядок 23). Цей метод повертає користувачеві стандартизований результат. Таким чином, код технічно обґрунтовує принцип «елегантної деградації» (graceful degradation), який дозволяє платформі Netflix продовжувати часткову роботу (наприклад, показувати базовий каталог замість персоналізованого), навіть під час збоїв у бекенді.
